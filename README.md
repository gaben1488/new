# Org Structure Diff Builder (Figma Plugin)

Production-ready плагин для Figma, который читает **две Excel-таблицы** (`1.xlsx` и `2.xlsx`) и строит оргструктуру с диффом изменений.

## Что реализовано

- Валидный `manifest.json` для импорта через **Import plugin from manifest**.
- UI с двумя загрузками Excel и кнопкой «Построить».
- Реальный Excel parser в UI через `xlsx` runtime:
  - чтение `xlsx/xls`;
  - обработка merged cells (`sheet['!merges']`);
  - наследование пустого отдела от предыдущей строки;
  - распознавание нестабильных заголовков столбцов;
  - парсинг ставок из числа/текста;
  - извлечение примечаний и флагов (декрет, замещение, совмещение, заместитель, распределено, ОУР).
- Slot-based модель в plugin runtime (`code.js`): отделы → слоты → назначения.
- Diff engine между текущей и будущей структурами:
  - `unchanged`, `added`, `removed`, `roleChanged`, `rateChanged`, `changedPerson`, `restructured`;
  - word-level diff должности (LCS по словам).
- Двухпроходный layout:
  - pass 1 — measure виртуального layout;
  - pass 2 — render в Figma-ноды.
- Отделы всегда в одну горизонтальную линию (без переносов).
- Аналитический блок + легенда.
- Детеминированные персональные цвета сотрудников по hash(name).

## Структура проекта

- `manifest.json` — манифест плагина.
- `ui.html` — интерфейс и Excel parser.
- `code.js` — runtime: модель, diff, layout, рендер.
- `excel_template.xlsx` — шаблон входной структуры.

## Установка

1. Откройте Figma Desktop.
2. `Plugins` → `Development` → `Import plugin from manifest...`
3. Выберите файл `manifest.json` из этого проекта.
4. Запустите плагин из `Plugins` → `Development`.

## Использование

1. В UI выберите:
   - текущую структуру (`1.xlsx`),
   - будущую структуру (`2.xlsx`).
2. Нажмите **Построить**.
3. Плагин создаст frame `Org Structure Diff` с:
   - заголовком,
   - аналитикой,
   - легендой,
   - колонками отделов в одну линию,
   - слотами сотрудников, ставками, вакансиями,
   - цветовым diff между состояниями.

## Самопроверка

Проверено перед коммитом:

- `manifest.json` валиден как JSON и содержит `api`.
- `code.js` проходит синтаксическую проверку.
- `ui.html` содержит два file input и кнопку построения, отправляет `build-org-chart` в runtime.
- В коде нет hardcoded sample rows для построения: построение идет из данных загруженных Excel.
- Реализована обработка `!merges` и наследование отдела.
- Отделы рендерятся в одну горизонтальную линию.

## Ограничения

- Внутри sandbox не выполняется реальный импорт в Figma Desktop, поэтому runtime проверяется статически и интеграционно по структуре/синтаксису.
- Загрузка `xlsx` runtime в UI идет из `https://unpkg.com`; домен добавлен в `networkAccess.allowedDomains`.
